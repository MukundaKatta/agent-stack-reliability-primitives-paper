# GitHub Maintainer Community + Stars Program — Application Bundle

Two adjacent GitHub recognition surfaces, both with a low click-cost. I can't submit on your behalf, but each block below is paste-ready.

## 1. Maintainer Community (low gate, do this first)

The Maintainer Community is GitHub's vetted private space for open-source maintainers. It runs as a request-to-join flow, not a multi-page application — GitHub vets you against your public footprint and grants access.

**Action:** open https://maintainers.github.com/, click "Login with GitHub", complete whatever short request form they show.

If the form asks for a paragraph about why you should be admitted, paste this:

> I maintain a public open-source AI reliability stack across npm, PyPI, and the Model Context Protocol registry. The agent-stack project ships eighteen packages — six libraries (AgentFit, AgentGuard, AgentSnap, AgentVet, AgentCast, AgentBudget), each with a TypeScript reference implementation, a Python port, and an MCP-server variant — under the `@mukundakatta` scope. All packages are MIT-licensed, zero-runtime-dependency, and have public test suites. The project is documented in a CC-BY-4.0 preprint on Zenodo (DOI 10.5281/zenodo.20074702) and in a peer-track submission at ASE 2026 Tools and Datasets. I also maintain Karna (a chat-native multi-channel personal AI assistant), AI Eval Forge (a mixed-check evaluation harness for LLM workflows), and roughly fifty other smaller libraries on npm and PyPI. Public profile: https://github.com/MukundaKatta. I'd value access to the maintainer community to learn from peers running similar AI-tooling projects and to participate in feature previews relevant to multi-language reliability libraries.

Maintainer Community gives you:
- Access to the private maintainer-only Discussions
- The Maintainer Partner Pack (deeper discounts than the public pack — Vercel, Algolia, JetBrains, etc.)
- Feature previews
- A small badge on your profile

## 2. GitHub Stars self-nomination (higher gate, do this second)

GitHub Stars is the higher-tier program. Form is short (one 180-char nomination statement), eligibility is "demonstrable impact on GitHub and the broader developer community in the past 12 months."

**Action:** open https://stars.github.com/nominate/, paste the values below.

| Field | Value |
| --- | --- |
| GitHub username of nominee | `MukundaKatta` |
| Nomination statement (≤180 chars) | `Maintains the agent-stack: 18 zero-dep AI reliability libs on npm/PyPI/MCP plus 9 preprints (Zenodo, Figshare, SSRN, Academia). 686 public repos.` (137 chars) |
| Acknowledgment checkbox | tick |

Three alternative 180-char statements to choose from:

1. *"Independent researcher shipping the agent-stack — 6 LLM-agent reliability libs across TypeScript, Python, MCP. CC-BY-4.0 papers on Zenodo. 686 public repos."* (162 chars)

2. *"Built and maintains the 18-package agent-stack on npm + PyPI + MCP registry. Open-source AI reliability primitives + 9 independent preprints with DOIs."* (158 chars)

3. *"Ships agent-stack: 6 zero-dep LLM reliability libs × 3 languages = 18 packages. Multiple Zenodo-archived preprints. Active on AI agent reliability."* (152 chars)

Pick whichever best matches your voice and self-attribution comfort. Statement 1 is the most balanced; statement 2 is most metric-heavy; statement 3 is most concise.

## What this gets you

Maintainer Community:
- ~5-min request, near-automatic acceptance with your footprint.
- Sponsor eligibility for the agent-stack libraries via Maintainer Partner Pack channels.
- Profile badge.

GitHub Stars:
- Application is a small lift; acceptance rate is materially lower (Stars cohort is ~100 globally per cycle).
- If accepted: visible badge, conference invitations, GitHub-paid feature previews, sponsor introductions.
- Self-nomination is cheap. Worst case the application is filed and not accepted; no downside.

## What I won't do

- Click submit for you. The Maintainer Community request and Stars nomination both go to GitHub under your name; you should review the values before they go in.
- Pad the nomination statement with claims that aren't true. The 180-char text above only references things publicly verifiable on your profile and on Zenodo.
